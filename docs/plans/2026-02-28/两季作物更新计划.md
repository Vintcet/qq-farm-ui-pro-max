# 两季作物兼容性更新计划

## 概述

本文档详细对比了 `qq-farm-bot-main 收两季作物` 项目与 `qq-farm-bot-ui-main` 项目的差异，重点分析**两季作物兼容性**相关的更新内容。

---

## 核心差异：两季作物处理逻辑

### 🎯 关键更新点：收获后重新检测土地状态

**位置：** `farm.js` - `checkFarm()` 函数

**问题背景：**
- 两季作物在收获第一季后，土地不会立即变为空地，而是继续生长第二季
- 旧版本在收获后直接将所有收获地块标记为"空地"并立即种植，导致第二季作物被误铲

**解决方案：**

```javascript
// 收获后重新检测土地状态，避免两季作物被误铲
if (harvestedLandIds.length > 0) {
    try {
        const refreshedReply = await getAllLands();
        if (refreshedReply.lands && refreshedReply.lands.length > 0) {
            const refreshedStatus = analyzeLands(refreshedReply.lands);
            for (const hid of harvestedLandIds) {
                if (refreshedStatus.empty.includes(hid)) {
                    allEmptyLands.push(hid);
                } else if (refreshedStatus.dead.includes(hid)) {
                    allDeadLands.push(hid);
                }
                // 仍在生长中（两季作物第二季）→ 不处理，等下次巡查
            }
        }
    } catch (e) {
        logWarn('巡田', `收获后刷新土地状态失败：${e.message}，跳过收获地块的后续处理`);
    }
}
```

**逻辑说明：**
1. 收获完成后，立即重新获取所有土地状态
2. 遍历已收获的地块 ID
3. 根据刷新后的状态判断：
   - 如果是**空地** → 加入种植队列
   - 如果是**枯死** → 先铲除再加入种植队列
   - 如果**仍在生长**（第二季）→ 不处理，等待下次巡查
4. 异常处理：刷新失败时记录警告，跳过后续处理

### 阶段一：核心逻辑更新（必须）

#### 1.1 修改 `runFarmOperation` 函数

**文件：** `/core/src/services/farm.js`

**修改位置：** 第 898-912 行（执行种植部分）

**当前代码：**
```javascript
// 执行种植
if (opType === 'all' || opType === 'plant') {

    const allEmptyLands = [...status.empty];
    // 注意：如果是单纯点"一键种植"，harvestedLandIds 为空，只种当前的空地/死地
    if (allDeadLands.length > 0 || allEmptyLands.length > 0) {
        try {

    }

```

**修改为：**
```javascript
// 执行种植
if (opType === 'all' || opType === 'plant') {
    let allDeadLands = [...status.dead];
    let allEmptyLands = [...status.empty];
    
    // 收获后重新检测土地状态，避免两季作物被误铲
    if (harvestedLandIds.length > 0) {
        try {
            const refreshedReply = await getAllLands();

                for (const hid of harvestedLandIds) {
                    if (refreshedStatus.empty.includes(hid)) {
                        allEmptyLands.push(hid);
                    } else if (refreshedStatus.dead.includes(hid)) {
                        allDeadLands.push(hid);
                    }
                    // 仍在生长中（两季作物第二季）→ 不处理，等下次巡查
                }
            }
        } catch (e) {
            logWarn('巡田', `收获后刷新土地状态失败：${e.message}，跳过收获地块的后续处理`);
        }
    }
    
    if (allDeadLands.length > 0 || allEmptyLands.length > 0) {
        try {
            const plantCount = allDeadLands.length + allEmptyLands.length;
            await autoPlantEmptyLands(allDeadLands, allEmptyLands, landsReply);
            actions.push(`种植${plantCount}`);
            recordOperation('plant', plantCount);
        } catch (e) { logWarn('种植', e.message); }
    }
}
```

**变更说明：**
- ❌ 删除：`const allDeadLands = [...status.dead, ...harvestedLandIds];`
- ✅ 新增：收获后重新检测逻辑
- ✅ 优化：先初始化 `allDeadLands` 为 `status.dead`，不直接包含 `harvestedLandIds`

- ✅ 支持普通肥料和有机肥

**建议：** 保留 ui-main 版本的自动解锁/升级功能（这是增强功能）

### D. 操作统计

**qq-farm-bot-ui-main:**

**建议：** 保留 ui-main 版本的操作统计功能（这是增强功能）
