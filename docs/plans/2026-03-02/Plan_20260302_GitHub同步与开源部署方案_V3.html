阶段一：代码纯净隔离与脱敏

  • 建立 ../qq-farm-ui-pro-max-github-sync 纯净物理隔离库。
  • 擦除 .git 及内部敏感缓存，包括 .env 文件内的 WX_API_KEY 与数据库口令。
  • 生成带有 MYSQL_HOST、REDIS_HOST、PORT 等强扩展性的 .env.example。

阶段二：ARM / x86_64 双架构原生 Docker 矩阵构建与推送

不再依赖外部缓慢工具，我们将首先在您本地直接进行交叉编译与推送：

  • 编写基于 node:22-alpine 的 Multi-stage Dockerfile。利用本地 Buildx 引擎构建超级压缩包（仅保留纯代码与依赖层）。
  • 一键构建并推送多架构命令 (我会在最终执行时带你运行)： 
    docker buildx create --use
docker buildx build --platform linux/amd64,linux/arm64 \
  -t smdk000/qq-farm-ui-pro-max:latest \
  -t smdk000/qq-farm-ui-pro-max:v3.8.0 \
  --push .

阶段三：高可用服务器一键部署编排 关键防丢策略

为小白服主提供完美整合 App + MySQL 8.0 + Redis 7.4 的 docker-compose.yml：

  • 数据完全持久化 mysql-data 挂载到 ./mysql_data，redis-data 挂载到 ./redis_data，应用配置挂载到 ./data。不论怎么 docker-compose down 都不丢任何东西！
  • ARM 容错设计： 若是甲骨文/树莓派 ARM ，因为 MySQL 8.0 本身限制，我们在配置文件中会智能利用 mysql:8.0-oracle 镜像或降维侦测，甚至通过 deploy.sh 放宽 vm.overcommit_memory 以保护 Redis。
  • 一键启动命令（供用户）： 
    curl -fsSL https://raw.githubusercontent.com/smdk000/qq-farm-ui-pro-max/main/deploy.sh -o deploy.sh && chmod +x deploy.sh && ./deploy.sh
    这个脚本会聪明地检测它身处 ARM 还是 x86_64 ，缺 Docker 就装 Docker ，然后一秒拉起三剑客（MySQL/Redis/Bot）。
