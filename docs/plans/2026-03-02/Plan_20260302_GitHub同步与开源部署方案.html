阶段一：代码隔离与彻底脱敏

  • 隔离 创建同级目录 ../qq-farm-ui-pro-max (避免误污染当前主开发目录)。
  • 清理 跳过所有 node_modules、dist、.env、.git 历史记录以及各种开发缓存。
  • 脱敏 扫描项目中的硬编码敏感信息（如遗留的真实密码、IP、QQ号、各种 API 的 AppId 等），全部通过 os.environ 提取。
  • 配置 生成规范的 .env.example 文档供开源用户参考。

阶段二：Docker 容器化与跨架构适配 (x86_64 & ARM64)

针对软硬机器（如苹果 M1/M2, 树莓派, 常用 Intel 服务器）：

  • 优化的 Dockerfile (Multi-stage Build): 采用交叉编译架构。构建出最小体积体积镜像，避免使用硬编码的平台依赖库。
  • 完善的 docker-compose.yml: 一次性整合主程序、Redis 和 MySQL/SQLite（若需）。
  • 健壮性设计: 添加 depends_on 和 healthcheck，指定容器日志轮转 (max-size: 10m) 以防撑爆服务器磁盘。

阶段三：CI/CD (GitHub Actions 自动发版)

  • 编写 .github/workflows/docker-publish.yml。
  • 触发机制: 配置在 pushing changes 到主分支，或发布 `tags` 时自动运行。
  • Buildx 矩阵: 采用 Docker Buildx 并行构建 linux/amd64 和 linux/arm64 并合并上传。
  • 智能缓存: 开启 type=gha 层缓存，极大提升后续 Github Actions 构建速度。

阶段四：保姆级一键部署脚本 (deploy.sh)

我将为您提供一个高度智能的 shell 脚本，即使用户只有一台空裸机，执行该脚本即可完成部署。
  • 自动侦测 Linux 操作系统，若未安装 Docker / Docker Compose 则自动通过清华镜像源安装。
  • ARM 专项优化: 针对小内存 ARM 云实例启动 Redis 常报错的问题，脚本自动执行 sysctl vm.overcommit_memory=1。
  • 自动检查端口占用，自动拉取 smdk000/qq-farm-ui-pro-max:latest 镜像并 up -d。
