🚀 策略流程编排 UI 重构方案

基于用户提供的竞品 UI 参考图，现有垂直两栏布局将彻底废弃，转为 单页横向流式编排 (Horizontal Track) 设计。此方案能更好地传达 "自动化序列" 的直观感受，且完全复用暗黑系 glass-panel 主题系统。

1. 视觉架构对比

🚫 旧版布局 (已废弃)

  • 双栏布局：左侧可用节点库，右侧流程队列。
  • 纵向排列：节点上下拖拽，占用垂直空间大。
  • 行内编辑独立：编辑面板固定在右侧容器，影响布局流。
  • 移动端割裂：在小屏幕上极其拥挤。

✅ 新版布局 (目标方案)

  • 单页横带：农场与好友流程分别为两个独立的上下文卡片。
  • 水平滚动轨道：节点横向排列，用 → 连接，更符合逻辑流。
  • 底部操作栏："添加节点工具栏" 结合 "保存操作按钮" 收纳于每个流程轨道底部。
  • 就地展开编辑：点击节点，在该节点下方横向展开编辑抽屉。

3. 技术实现细节

  • 拖拽引擎变更：保留 Native Pointer Event 物理引擎，但数学计算从 Y轴 (Vertical) 改为 X轴 (Horizontal) 测量坐标。
  • CSS 响应式：横向轨道使用 white-space: nowrap 和 overflow-x: auto 搭配 custom-scrollbar 确保内容可滑，不卡屏幕。
  • 动态节点宽度：节点为 inline-flex 撑开宽度，Ghost 占位同样按真实宽度计算。
  • 展开参数面板 (Inline Panel)：若某一节点带参数，点击时在该 Container 内部下方展开一栏面板。使用 Vue v-if 挂载在该行下方即可。

4. 影响面评估

  • 前端修改范围：只重构 Workflow.vue，不增加新依赖。
  • 后端数据结构：依然序列化为 { "uuid", "type", "params" } 的列表，兼容之前版本或空状态。
  • 交互体验：保留用户全局的 Glassmorphism 样式。在低配设备或性能模式下依然流畅工作。
