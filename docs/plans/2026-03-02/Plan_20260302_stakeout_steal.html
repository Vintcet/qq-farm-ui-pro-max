🚀 增强计划：蹲守偷菜模式 (Stay-And-Steal)

概述： 当前的自动偷菜受限于固定间隔的轮询扫描，容易在下一次轮询到来前被其他玩家“秒偷”。本方案旨在实现精准预判与定时触达：在日常扫田时记录好友农场所有作物的成熟倒计时，建立“提前上线踩点 -> 无感等待 -> 成熟后防风控延迟 -> 精准收割”的智能调度机制。

🧩 核心数据流转

⚙️ 架构变更详述

1. 前端 UI (Settings 面板优化)

在目前的 StealSettings.vue 增加 “高级偷菜与蹲守策略” 控制区：

  • stakeoutEnabled：控制是否进行潜伏式蹲点。
  • stakeoutDelaySec：控制作物成熟后，等待多少秒再发起收割动作。支持最小 1 秒，以对抗防外挂机制防检测零秒秒偷。

数据通过既有框架同步到 localSettings.automation 并落盘云端，无破坏性后端表结构变更。

2. 核心分析增强 (analyzeFriendLands)

目前 friend.js -> analyzeFriendLands 只分析 MATURE 和 DEAD 状态。
增加对 GROWING 期且即将成熟（有 matureInSec > 0）作物的提取，并加入返回值 upcomingMature 数组中。
规则保障： 在提取时同样先经过黑名单与白名单判定（shouldStealPlant / shouldStealFriend），避免无效调度。

3. 智能定时器调度 (Smart Scheduler)

在现有系统中的 scheduler.js (使用 setTimeoutTask) 中创建和管理定时任务：

sequenceDiagram participant Loop as 巡查循环 participant Mem as 内存调度堆栈 participant Execution as 执行引擎 Loop->>Mem: 发现好友A有地块(1,2) 延时50s成熟 Mem-->>Mem: 覆盖或建立 Task(stake_A, delay: 50s-3s) Mem-->>Execution: 等待 47s Execution->>Execution: [踩点进入] await enterFriendFarm() Execution->>Execution: [安全防风控延迟] await sleep(3 + 设定延迟s) Execution->>Execution: 发起偷取请求 stealHarvest()并记录

处理同好友多地块合批： 为了防御大量地块接近同时成熟导致多次请求被封，当多个地块成熟时间差在 5~10 秒内 时，进行同组归并，由一个定时蹲守一并处理。

💥 影响面分析 (Impact Analysis)

模块 影响面 风险与缓解方案
API 限流防封 增加并发的 enterFarm 和 harvest 请求。 已经有全局 Token 桶控制 sendMsgAsync；且蹲守任务只对有效过滤作物品发力，并与普通巡田做时序隔离（踩点后 Sleep 时挂起当前 Worker，此时其他队列在等候）。
UI 与配置中心 只增加新属性，不会覆盖或造成老属性失效。 Object.assign 或者解构赋值完美向后兼容，若原本没有 stakeout_steal 默认为 false。
内存与开销 会增加一部分 Node.js setTimeout 句柄。 设定最长生效窗口期（如 4 小时），如果 4 小时还不成熟就不注册 timer（或等下次扫描重新注册），能充分减少死 Timer 对象滞留。
行动指引： 本计划无需修改数据库表结构，将对 core/src/services/friend.js 及 web/src/views/StealSettings.vue 进行非侵入性扩展。所有行为通过用户手工配置开关管控。请审阅可行性与逻辑严密性后继续执行 Atomize 和 Automate 动作。
