撰写时间：2026-03-02 | 目标：打破硬编码流水线框架，构建用户侧动态节点挂载引擎

🚀 背景与业务破局点

目前农场的调度引擎（`worker.js` -> `farm.js`）采用了极强耦合的顺序 await 请求组装模式。虽然稳定，但丧失了**“灵活性”**与**“玩家战术配置”**的空间。例如用户提出希望能在作物“仅剩几分钟成熟”时拉起等待防偷锁闭动作，传统的 `if...else` 堆叠已经难以优雅支撑这类诉求。

引入 可视化编排 (Drag-and-Drop Workflow Engine) 是破局唯一解。将“固化代码”升维为“JSON 字典解释器”。

🛠️ 核心域解构详解

1. 运行时状态记忆库扩展

必须在 core/src/models/store.js 的底座基础数据模型中插入节点数组寄存器，以承接前端吐回的拓扑节点数据。

// Data Schema Demo
workflowConfig: {
    farm: {
        enabled: true,
        nodes: [
            { id: "uuid-1", type: "STAGE_FERTILIZE", props: { stages: ['seed', 'sprout'] } },
            { id: "uuid-2", type: "WEED" },
            { id: "uuid-3", type: "WAIT_MATURE", props: { stopRestIfUnmature: true } }
        ]
    }
}

2. 引擎驱动器开发 (Interpreter Pattern)

此模块属于风险重点。需在 farm.js 中新增独立的流水线承接函子，该函子将依次 switch (node.type) 并派发给细粒度的真实通信动作。在此过程中，特别要留意“等待成熟中止后续操作”所带来的栈挂起或者回溯中断抛出处理。

🎨 Web 前端实现要略

  • 模块加载：锁定引入轻量原生的 vuedraggable@4.1.0。
  • 视效切地：重开一路由，或提供“独立弹窗”，以双区展示：左侧（资源面板 / Node Pool），右侧（泳道 / Operation Lane）。
  • 参数配置表盘：采用事件总线点击节点，弹出右侧或底部的 Drawer 配置独立节点的参数（避免挤压拖拽轨道面积）。

🛡️ 风险与安全制约边界

  • 死板兼容：不能破坏旧版全局自动开关的功能。两者需要互斥保护。
  • 内存渗漏熔断：必须硬编码写死「最大延时休眠时长」和「单次流程最大执行周期」，防止玩家配置无意义阻塞链条，烧毁有限的后台异步请求分发池。
