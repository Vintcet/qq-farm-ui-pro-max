任务 1：API Key 迁移至环境变量

🔒 安全 高

问题描述

core/src/controllers/admin.js 中微信扫码 API 的密钥 api_key: 'ZoBsM8RqTA326a02tWgQOmOb' 硬编码出现在 3 个位置（第 1072、1107、1115 行）。 硬编码密钥存在以下风险：

⚠️ 影响面：若代码泄露（如公开仓库推送），API Key 会被直接暴露，可能被滥用导致服务封禁或产生费用。 此外，如需更换 Key，必须修改源代码并重新部署，缺乏运维灵活性。

实施方案

  1. 在项目根目录的 .env 文件中新增变量 WX_API_KEY=ZoBsM8RqTA326a02tWgQOmOb
  2. 在 .env.example（如存在）中添加占位 WX_API_KEY=your_wx_api_key_here
  3. 确认 .gitignore 已包含 .env（防止误提交）
  4. 修改 admin.js 顶部，添加 const WX_API_KEY = process.env.WX_API_KEY || '';
  5. 将 3 处 api_key: 'ZoBsM8RqTA326a02tWgQOmOb' 替换为 api_key: WX_API_KEY
  6. 在启动入口（如 index.js）确认 dotenv 已加载（require('dotenv').config()），若项目未使用 dotenv 模块，则通过 CONFIG 对象读取

涉及文件

  • core/src/controllers/admin.js — 替换 3 处硬编码 api_key
  • .env — 新增 WX_API_KEY 变量
  • .gitignore — 确认包含 .env
注意事项：需检查项目启动时是否已有 dotenv 加载逻辑。如果项目使用自有的 CONFIG 配置体系（如 core/src/config/config.js），可优先考虑将 Key 挂载于 CONFIG 对象上保持一致性。

任务 2：NProgress 进度条主题色联动

✨ 增强 中

问题描述

web/src/style.css 第 46-55 行，NProgress 进度条颜色硬编码为 #10b981（emerald-500）。 当用户切换至 Cyber（紫色）、Sakura（粉色）、Ocean（蓝色）等主题时，顶部加载进度条仍保持绿色，与整体视觉风格不协调。

⚠️ 影响面：纯视觉体验问题，不影响功能。但在多主题架构下属于"漏网之鱼"，会降低主题切换的完整性和高级感。

实施方案

  1. 将 #nprogress .bar 的 background 从 #10b981 改为 rgb(var(--color-primary-500))
  2. 将 #nprogress .peg 的 box-shadow 中的两处 #10b981 也改为 rgb(var(--color-primary-500))

涉及文件

  • web/src/style.css — 修改 2 个 CSS 规则（约 4 行）
